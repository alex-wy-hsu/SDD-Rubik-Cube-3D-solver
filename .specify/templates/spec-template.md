### 非功能需求（憲章規定）

#### 代碼品質需求
- 所有代碼必須遵循單一職責和 DRY 原則
- 公共 API 必須有文檔
- 代碼必須通過 linting 且零警告

#### 測試需求（不可妥協）
- 所有功能必須在實作前編寫測試（TDD）
- 單元測試覆蓋率必須 ≥80%
- 所有用戶故事必須有整合測試
- 測試必須涵蓋邊界情況和錯誤條件

#### 用戶體驗需求
- 視覺元素必須遵循一致的渲染慣例
- 用戶操作必須提供即時反饋（<200ms 或載入指示器）
- 錯誤訊息必須清晰且可操作
- UI 必須支援鍵盤導航

#### 性能需求
- 3D 渲染在動畫期間必須維持 ≥60 FPS
- 方塊狀態驗證必須在 <10ms 內完成
- 解法生成必須在 <5 秒內完成
- 記憶體使用必須保持在 200MB 以下
- 初始載入必須在 <3 秒內完成
